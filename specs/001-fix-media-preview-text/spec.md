# Feature Specification: Fix Media Preview Text

**Feature Branch**: `main`
**Created**: 2026-05-30
**Status**: Ready for planning
**Input**: Blog cards show media-derived text such as `1.00` or `Image` before the real excerpt.

## User Scenarios & Testing

### User Story 1 - Read meaningful blog previews (Priority: P1)

As a reader scanning blog cards, I want the excerpt to start with the article's written content, not image labels or media metadata, so that I can understand what the blog is about before opening it.

**Why this priority**: The issue appears on public blog cards and makes the site look broken whenever a post starts with media.

**Independent Test**: Given a blog whose content starts with an image followed by prose, the visible card excerpt starts with the prose and does not include `Image`, `1.00`, or other media-only labels.

**Acceptance Scenarios**:

1. **Given** a blog starts with a markdown image whose alt text is `Image`, **When** the archive card renders its excerpt, **Then** the excerpt starts with the first prose sentence after the image.
2. **Given** a blog starts with a markdown image whose alt text is `1.00`, **When** the archive card renders its excerpt, **Then** the excerpt does not display `1.00`.
3. **Given** a blog contains inline links and ordinary text, **When** preview text is generated, **Then** link labels remain readable text.
4. **Given** a blog contains only media and no prose, **When** preview text is generated, **Then** the existing fallback preview is shown.

## Requirements

### Functional Requirements

- **FR-001**: Preview text generation MUST exclude markdown image syntax and image-only labels from excerpts.
- **FR-002**: Preview text generation MUST exclude HTML image tags from excerpts.
- **FR-003**: Preview text generation MUST preserve ordinary link labels as readable preview text.
- **FR-004**: Blog archive and featured-card reading-time calculations MUST ignore media-only syntax and metadata.
- **FR-005**: Existing fallback preview behavior MUST continue when no readable prose remains.
- **FR-006**: The fix MUST not change image selection for card covers.

### Key Entities

- **Blog content**: The canonical markdown body used for excerpts, reading time, and cover extraction.
- **Preview text**: Plain text derived from blog content for cards, hero previews, profile lists, and archive summaries.
- **Media block**: Markdown or HTML image content that may carry generic labels or editor metadata.

## Success Criteria

- **SC-001**: A markdown sample beginning with `![Image](...)` produces preview text without `Image`.
- **SC-002**: A markdown sample beginning with `![1.00](...)` produces preview text without `1.00`.
- **SC-003**: A markdown sample beginning with an HTML image tag produces preview text without image attributes.
- **SC-004**: Lint and TypeScript validation pass for the frontend repository using a supported Node runtime.

## Assumptions

- Image alt text generated by the editor is not meaningful enough for card excerpts.
- Cover image extraction remains separate from preview text extraction.
- This is frontend-only; no backend API or stored content migration is required.
